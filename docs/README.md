# Filing System Documentation

This documentation provides a comprehensive guide to the filing system, including the folder structure, a flowchart for deciding where to file documents, and a universal file naming convention.

## How to Use This Documentation

*   **`index.html`**: This is the main page of the documentation, containing the filing flowchart and naming convention. It's designed to be used as a static website.
*   **`README.md`**: This file provides a brief overview of the documentation.

## Deployment

This documentation is designed to be deployed as a static website. You can use any static site hosting provider, such as GitHub Pages, Netlify, or Vercel. Simply point the hosting provider to the `docs` directory.
